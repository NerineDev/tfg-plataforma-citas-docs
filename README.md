# Appointment Management Platform – Project Docs

Public documentation repository for my **Proyecto Integrado** (Final Project) of the *Desarrollo de Aplicaciones Web (DAW)* cycle.

This repo stores:

- The official deliveries for the module (E0, E1, E2, E3).
- Diagrams and early design sketches.
- Notes, decisions and planning related to the project.

> 📄 **Spanish version**: see [`README.es.md`](README.es.md)

---

## Project overview

The project is a **web platform for appointment and service management** aimed at freelance professionals and small businesses.

The goal is to provide:

- An online booking system for clients.
- A simple control panel for professionals (availability, services, calendar).
- Integrated online payment for sessions (test mode during the TFG).

The **source code** of the application will live in a **private repository** to protect the project’s intellectual property.  
This public repo is only for documentation and planning.

## Repository structure

```
/entregas
    PI_E0, PI_E1, PI_E2, PI_E3...

/bocetos
    Architecture diagrams, use–case diagrams, wireframes, UI sketches...

/notas
    Backlog, roadmap, changelog, decisions and other working notes.
```

### `/entregas`

Folder with the official deliveries of the Proyecto Integrado module:

- `PI_E0` – Initial project choice and description. *(Submitted.)*
- `PI_E1` – Specification and analysis. *(Submitted.)*
- `PI_E2` – Implementation and development. *(Submitted.)*
- `PI_E3` – Deployment, testing and final report. *(Submitted.)*

### `/bocetos`

Wireframes, architecture diagrams, navigation maps and any visual material used to plan the project.

### `/notas`

Working documents used to organize the development:

- `backlog.md` – Feature and task backlog.
- `roadmap.md` – High–level project phases and milestones.
- `changelog.md` – Log of relevant changes in the documentation.
- `decisiones.md` – Design and technical decisions.

These files are **living documents** and can change as the project evolves.
